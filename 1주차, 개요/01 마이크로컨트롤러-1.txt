마이크로 컨트롤러.
마이크로 프로세서와의 차이?

CPU : 연산장치 + 제어장치.
주기억장치.
위의 2가지는 메인보드에 장착이 된다.

마이크로 프로세서 : 컴퓨터의 중앙처리장치(CPU)를 단일 IC칩에 집적해 만든 반도체소자.
단순히 프로세서 혹은 MPU(Micro Processer Unit)라고도 부름.

마이크로컴퓨터 : 마이크로프로세서와 기억장치, 입출력장치 등을 모아 하나의 시스템으로 구성한 것
소형, 경량화, 저소비전력, 저가, 부품수 감소로 인한 신뢰성 향상

마이크로컨트롤러 : 단일 칩 내 마이크로프로세서의 CPU코어와 함께 한정된 용량의 주기억장치, 입출력인터페이스를 내장
하나의 칩 내에 내장된게 핵심.
칩 하나만으로 기본적 컴퓨터의 기능 수행.
IC소자 하나로 완전한 형태의 컴퓨터가 구성되므로 단일칩 마이크로컴퓨터라고도 부른다.

정리)
마이크로프로세서 : 컴퓨터의 중앙처리장치를 하나의 칩으로 구현.
마이크로 프로세서를 중심으로 ROM, RAM, 입출력장치등을 추가하여 마이크로컴퓨터를 구성함.

마이크로 컨트롤러 : 마이크로컴퓨터에 들어가는 모든 부품들을 하나의 반도체 칩에 집적한 것으로 One chip mircocomputer.
마이크로 프로세서의 일종!

소형, 경량화가 가능하다.
가격이 저렴하다
타 시스템과의 이식성이 뛰어나다 : 대부분의 제어 기능이 하나의 IC에 내장되어 있으므로 시스템을 변경하고자 할 경우에는 약간의 기능 변경과 추가를 통해 쉽게 달성할 수 있어 다양한 용도로 활용이 가능함.
신뢰성이 높다 : 시스템 구성 소자수가 적어 제품의 신뢰성이 높다.

특정기기를 제어하는데 많이 사용된다.

마이클로컨트롤러를 위한 프로그램 개발과정
교차 개발 과정
프로그램 개발은 개발 시스템(데스크톱 컴퓨터)에서 진행
개발 시스템은 목적 시스템(마이크로컨트롤러)에서 실행되는 기계어 파일을 생성할 수 있는 교차 컴파일러 사용
개발된 기계어 파일은 전용 장치를 사용하여 목적 시스템으로 업로드.

프로그램 업로드 방식
ISP 방식(In System Programming) 방식.
SPI 시리얼 통신 사용
마이크로 컨트롤러의 메모리에 직접 프로그램 기록.
AVR 마이크로컨트롤러에서 주로 사용하는 방식.

시리얼(Serial) 방식.
UART 시리얼 통신 사용
부트로더를 통해 마이크로 컨트롤러의 메모리에 프로그ㅐㄻ 기록.
아두이노에서 주로 사용하는 방식.

EX) 점등 회로의 경우
마이크로 컨트롤러 사용의 장단점
단점 : 점등 회로의 경우 하드웨어로만 구현하는 방식에 비해 장점 없음
마이크로 컨트롤러 기능을 100% 사용하지 않음.
별도의 소프트웨어(펌웨어)를 구현해야 한다.
장점 : 기능을 변경 혹은 추가하는 경우 동일한 입출력을 사용한다면 소프트웨어 변경만으로 간으.
무어의 법칙에 따라 마이크로컨트롤러의 성능은 계속 발전하고
가격은 지속적으로 하락하고 있어 기능을 100% 사용하지 않는 경우에도 소형화, 경량화, 초기 개발 시간 단축. 유지보수 비용 절감 등의 장점으로 경쟁력 확보 가능.

무어 머신: 출력이 현재의 상태에서만 의존.
밀리 머신: 출력이 현재의 상태, 입력에 의해 영향.
무어의 법칙 : 24개월 마다 칩안의 트랜지스터 개수가 2배가 된다
